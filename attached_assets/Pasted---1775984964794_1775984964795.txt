مزامنة الإشعارات بين جميع الأجهزة: 🔔 أولاً: نظام الإشعارات الفكرة الأساسية النظام مكوّن من 3 طبقات تعمل معاً في آنٍ واحد عند كل حدث مهم: حدث (مثلاً: موافقة على تقسيط) ↓ fireNotify() ←── نقطة الدخول الوحيدة / | \ ↓ ↓ ↓ DB WS Push الطبقة 1 — قاعدة البيانات (MongoDB) // يحفظ الإشعار في NotificationModel حتى لو الجهاز مطفي await NotificationModel.create({ userId, type: "success", title: "✅ تمت الموافقة", body: "...", link: "/installments", icon: "✅" }); الغرض: الإشعار يبقى موجوداً ويظهر في صفحة "الإشعارات" لاحقاً. الطبقة 2 — WebSocket (اللحظي) // يرسل للمستخدم لو الجهاز مفتوح الآن pushToUser(userId, { type: "notification", title, body, link }); الغرض: تظهر نافذة منبثقة فورية داخل التطبيق بدون تحديث الصفحة. الطبقة 3 — Web Push (خارج التطبيق) // يرسل إشعار نظام حتى لو التطبيق مغلق await sendPushToUser(userId, { title, body, icon: "/icons/icon-192x192.png", tag: "notif-xyz", // يمنع تكرار الإشعار data: { url: "/installments" } }); الغرض: يظهر كإشعار نظام على الهاتف أو الحاسوب حتى لو المستخدم خارج التطبيق كلياً. كيف يعمل Web Push تقنياً (VAPID) المتصفح يسجّل اشتراك Push → يُحفظ في PushSubscriptionModel endpoint + keys (p256dh + auth) ↓ السيرفر يرسل للـ endpoint عبر بروتوكول Web Push ↓ خادم المتصفح (Google/Apple/Mozilla) يرسل للجهاز إذا انتهت صلاحية الاشتراك (خطأ 410/404) يُحذف تلقائياً من قاعدة البيانات. الدوال المساعدة الدالة الاستخدام fireNotify(userId, title, body, opts) إشعار مستخدم واحد بعينه fireNotifyAdmins(title, body, opts) إشعار كل المديرين والمشرفين دفعة واحدة
